# はじめてガイド（だれでも分かる使い方）

このシステムは、こういうものです👇

> **現場の壁にQRコードをはる → スマホでピッと読みとる → 注意することがスマホに出てくる**

ポスターを貼るかわりに、QRコードを貼るイメージです。
むずかしい操作はありません。使う人は3タイプに分かれます。

---

## 🧑‍🏭 タイプ1：見る人（現場ではたらく人）

**やることは1つだけ。スマホでQRコードを読むだけ。**

1. 現場の壁にはってあるQRコードを見つける
2. スマホの **カメラアプリ** をひらいて、QRコードにかざす
3. 画面に出てきた青いリンクをタップ
4. その場所の「気をつけること」が出てくる ✅

> アプリのインストールはいりません。ログインもいりません。
> LINEのQRコードを読むのと同じ感じです。

---

## ✏️ タイプ2：内容を直す人（担当者）

「ここの説明を変えたい」「注意を1つ追加したい」というときの人です。

### イメージ
内容は **`areas`（エリア）というフォルダの中のメモ帳ファイル** に書いてあります。
そのメモ帳の文字を書きかえるだけです。

### かんたんな例

```
"title": "工具棚の整理",          ← この「" "」の中の文字を直す
"description": "使った後は元の場所に戻す"   ← ここも直すだけ
```

### 気をつけるルール（これだけ守ればOK）
- `"` （ダブルクォーテーション）の **中の文字** だけを直す
- `{ } [ ] " ,` の記号は **消したり増やしたりしない**（こわれます）
- 直したら、日付（`"updated"`）と名前（`"updatedBy"`）も今日のものに直す

> 重要度は3段階から選べます。
> - 🔴 `critical`（クリティカル）… ケガ・事故にかかわる超重要なこと（自動で一番上に出る）
> - 🟠 `high`（ハイ）… 品質や効率にかかわること
> - ⚪ `normal`（ノーマル）… ふつうの注意

くわしくは [更新ガイド](update-guide.md) を見てください。

---

## 🖨️ タイプ3：QRコードを作る・はる人（管理者）

新しい場所のQRコードを作って、現場にはる人です。

### 手順
1. パソコンで **`qr-generator.html`** をひらく
2. 上のらんに「公開URL」を入れる
   （例：`https://〇〇.github.io/5s-app/`。分からなければ作った人に聞く）
3. はりたい現場をえらぶ
4. 「🖨 印刷する」ボタンをおす → A4の紙に印刷される
5. **はる前に、自分のスマホで読みとってテスト**（だいじ！）
6. 問題なければ、よく見える高さ（床から120〜150cmくらい）に貼る

> 内容（注意ごと）を後から書きかえても、**QRコードは貼りなおさなくてOK**です。
> QRコードは「住所」を指しているだけで、中身が変わっても住所は同じだからです。

くわしくは [QRコード印刷ガイド](qr-print-guide.md) を見てください。

## ❓ こまったとき

| こまったこと                         | どうする                                       |
| ------------------------------------ | ---------------------------------------------- |
| QRを読んでも何も出ない               | 印刷がかすれていないか・URLが正しいか確認       |
| 「現場が見つかりません」と出る        | QRコードか、ファイルの設定が合っているか確認    |
| 「読み込みに失敗しました」と出る      | メモ帳ファイルの記号（`{ } " ,`）がこわれてるかも |
| 画面がへんな表示になる               | 直す前にもどす／作った人に聞く                  |

---

## 3行でまとめ

- **見る人**：スマホのカメラでQRをピッ → 注意が出る
- **直す人**：メモ帳ファイルの文字を書きかえるだけ（記号はさわらない）
- **作る人**：`qr-generator.html` で印刷して、テストしてから貼る
