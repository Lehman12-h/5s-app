# コンテンツ更新ガイド（担当者向け）

このガイドは、プログラミングの知識がなくても5S活動の掲示内容を更新できるように作成しています。
編集するのは **`areas/` フォルダの中の JSON ファイル** だけです。

---

## 1. はじめに知っておくこと

- 1つの現場 = 1つのファイル（例：`areas/area-A1.json`）
- ファイルは「メモ帳」や「VS Code」などのテキストエディタで開けます
- 記号（`{` `}` `[` `]` `"` `,`）はルールがあるので、**消したり増やしたりしない**でください
- 変更を保存して GitHub にアップロードすると、約1分で公開ページに反映されます

---

## 2. 既存の注意事項を直したいとき

### 手順

1. `areas/` フォルダから、直したい現場のファイルを開く（例：`area-A1.json`）
2. 文章を直したい項目の `"title"`（件名）や `"description"`（説明）の
   **ダブルクォーテーション `"` の中** の文字だけを書き換える
3. ファイルの上のほうにある `"updated"` を今日の日付に直す（例：`"2026-06-11"`）
4. `"updatedBy"` を自分の名前に直す（例：`"安全管理 鈴木"`）
5. 保存して GitHub にアップロード（→ 4章へ）

### 編集してよい場所の例

```json
{
  "id": "A1-001",
  "title": "工具棚の整理",                ← ここを書き換えてOK
  "priority": "high",                      ← critical / high / normal から選ぶ
  "description": "使用後は必ず…",          ← ここを書き換えてOK
  "image": "images/area-A1-shelf.jpg",     ← 画像を変えるとき（5章）
  "imageAlt": "工具棚の正しい整理例",       ← 画像の説明
  "tags": ["整理", "整頓"]                 ← タグ（任意）
}
```

---

## 3. 注意事項を1つ追加したいとき

既存の項目を **`{` から `}` までまるごとコピー** して、`,`（カンマ）でつなげます。

### 例：1つ追加する場合

```json
  "items": [
    {
      "id": "A1-001",
      "title": "工具棚の整理",
      "priority": "high",
      "description": "使用後は必ず所定の位置に戻すこと。",
      "tags": ["整理", "整頓"]
    },                                       ← ここにカンマを付ける
    {
      "id": "A1-099",                         ← 番号は重複しないように
      "title": "新しい注意事項",
      "priority": "normal",
      "description": "ここに内容を書きます。",
      "tags": ["清掃"]
    }                                         ← 最後の項目にはカンマを付けない
  ]
```

> **ポイント**
> - 項目と項目の間には `,`（カンマ）を入れる
> - **一番最後の項目の後ろにはカンマを入れない**
> - `id` は他とかぶらない番号にする（例：`A1-001`, `A1-002`…）

---

## 4. 変更を公開ページに反映する（GitHub へのアップロード）

> 会社のルールに合わせて、いずれかの方法で行ってください。

### 方法A：GitHub の画面から直接編集（いちばん簡単）

1. ブラウザで会社の GitHub リポジトリを開く
2. `areas/` フォルダ → 編集したいファイルをクリック
3. 鉛筆マーク（✏ Edit）を押す
4. 内容を書き換える
5. ページ下部の **「Commit changes」** ボタンを押す
6. 約1分待つと公開ページに反映されます

### 方法B：パソコンのファイルを編集して同期

担当のエンジニアの手順に従ってください（`git push`）。

---

## 5. 画像を追加・差し替えたいとき

詳しくは [`images/README.md`](../images/README.md) を参照してください。要点は次の通りです。

1. 画像を `images/` フォルダに入れる
2. ファイル名は **半角英数字** にする（例：`area-A1-shelf.jpg`）
3. JSON の `"image"` にパスを書く：`"image": "images/area-A1-shelf.jpg"`
4. `"imageAlt"` に画像の説明（日本語）を書く
5. 画像は **1枚あたり 500KB 以下** にする（大きいと表示が遅くなります）

画像が無い項目は、`"image"` を `""`（空）にするか、行ごと消してかまいません。

---

## 6. priority（重要度）の使い分け

| 値          | 表示          | 使う場面                       |
| ----------- | ------------- | ------------------------------ |
| `critical`  | 🔴 ⚠ 重要     | ケガ・事故につながる安全項目   |
| `high`      | 🟠 注意       | 品質・効率に影響する項目       |
| `normal`    | （ラベルなし）| 一般的な注意事項               |

`critical` の項目は、自動で画面の **いちばん上** に表示されます。

---

## 7. やってはいけないこと

- 記号 `{ } [ ] " ,` を消す・増やす（表示できなくなります）
- ファイル名を日本語にする
- `id` を他の項目とかぶらせる

---

## 8. うまく表示されないとき

- 画面に「読み込みに失敗しました」と出る → JSON の記号が崩れている可能性大。
  直前の編集を見直してください。
- [JSONLint（無料の文法チェッカー）](https://jsonlint.com/) に貼り付けると、
  どこが間違っているか教えてくれます。

---

困ったときは、システム担当のエンジニアに JSON ファイルを見てもらってください。
