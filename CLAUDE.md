## 1. プロジェクト概要

### 目的
工場・現場の5S活動（整理・整頓・清掃・清潔・躾）における注意箇所を、各現場に掲示したQRコードからスマホで閲覧できるシステムを構築する。

### 要件
- **閲覧者**: 約100人（同時アクセスを想定）
- **閲覧方法**: QRコードをスマホカメラで読み取りブラウザで表示（専用アプリ不要）
- **コスト**: 初期・運用ともに0円（完全無料）
- **更新**: 管理者がファイルを編集するだけで即時反映
- **コンテンツ**: テキスト＋イラスト（注意箇所の画像・写真）

### 採用スタック
| レイヤー | 技術 | 理由 |
|----------|------|------|
| ホスティング | GitHub Pages | 無料、CDN配信、100人同時閲覧対応 |
| フロントエンド | HTML / CSS / Vanilla JS | ビルド不要、スマホ最適化しやすい |
| データ管理 | JSON ファイル | 非エンジニアでも編集可能 |
| QRコード | URLから静的生成 | 外部サービス不要 |
| バージョン管理 | Git / GitHub | 更新履歴保持、ロールバック可能 |

## 3. データスキーマ

### 3-1. 現場JSONファイル（`areas/area-XX.json`）

```json
{
  "id": "area-A1",
  "name": "第1製造ライン A工程",
  "location": "第1工場 1F 東側",
  "updated": "2026-06-01",
  "updatedBy": "安全管理 山田",
  "category": "製造",
  "items": [
    {
      "id": "A1-001",
      "title": "工具棚の整理",
      "priority": "high",
      "description": "使用後は必ず所定の位置に戻すこと。棚の上に物を置かない。",
      "image": "images/area-A1-shelf.jpg",
      "imageAlt": "工具棚の正しい整理例",
      "tags": ["整理", "整頓"]
    },
    {
      "id": "A1-002",
      "title": "通路の確保",
      "priority": "critical",
      "description": "黄色ラインより内側に物を置かないこと。緊急時の避難経路を常に確保する。",
      "image": "images/area-A1-passage.jpg",
      "imageAlt": "通路確保の基準線",
      "tags": ["安全", "整頓"]
    }
  ]
}
```

### フィールド定義

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `id` | string | ✅ | `area-` + 英数字。ファイル名と一致させる |
| `name` | string | ✅ | 現場の正式名称 |
| `location` | string | ✅ | 建屋・フロア・エリア |
| `updated` | string | ✅ | ISO 8601形式 `YYYY-MM-DD` |
| `updatedBy` | string | ✅ | 最終更新者（所属 氏名） |
| `category` | string | ✅ | 製造 / 倉庫 / 事務 / 共用 のいずれか |
| `items[].id` | string | ✅ | `{エリアID}-{3桁連番}` 形式 |
| `items[].title` | string | ✅ | 注意事項の件名（20文字以内推奨） |
| `items[].priority` | string | ✅ | `critical` / `high` / `normal` のいずれか |
| `items[].description` | string | ✅ | 具体的な指示（200文字以内推奨） |
| `items[].image` | string | — | 画像パス。なくてもよい |
| `items[].imageAlt` | string | 画像あり時 | スクリーンリーダー用代替テキスト |
| `items[].tags` | array | — | 整理 / 整頓 / 清掃 / 清潔 / 躾 / 安全 |

### priority の表示仕様
- `critical`：赤バッジ「⚠ 重要」— 安全に直結する項目
- `high`：オレンジバッジ「注意」— 品質・効率に影響する項目
- `normal`：グレーバッジ（ラベルなし）— 一般的な注意事項

## 4. URLルール

```
https://{GitHubユーザー名}.github.io/5s-app/?area=area-A1
```

- `area` パラメータで現場JSONのIDを指定する
- パラメータ省略時はエリア一覧ページを表示する
- 存在しないIDを指定した場合は「現場が見つかりません」エラー画面を表示する

### QRコード生成URL
各現場のURLは `admin.html` または `qr-generator.html` から一覧生成できる。
印刷用のA4レイアウト（現場名・URL・QRコード入り）も同ページから出力する。

## 5. UI設計ルール

### 5-1. モバイルファースト
- ビューポート幅: 375px（iPhone SE）を基準に設計
- タッチターゲット: 最小44×44px
- フォントサイズ: 本文16px以上、見出し20px以上
- 画像: `max-width: 100%`、遅延読み込み（`loading="lazy"`）

### 5-2. 表示構成（スマホ閲覧画面）
```
┌─────────────────────────┐
│ [会社ロゴ]  5S活動      │  ← ヘッダー（固定）
│ 第1製造ライン A工程     │
├─────────────────────────┤
│ 📍 第1工場 1F 東側      │  ← 場所・更新日
│ 更新: 2026-06-01 山田   │
├─────────────────────────┤
│ ⚠ 重要  通路の確保     │  ← priorityバッジ + タイトル
│ [写真]                  │
│ 黄色ラインより内側に…   │  ← 説明文
├─────────────────────────┤
│ 注意  工具棚の整理      │
│ [写真]                  │
│ 使用後は必ず所定の…     │
└─────────────────────────┘
```

### 5-3. 禁止事項（UI）
- ログイン・認証は一切実装しない（QRで誰でも閲覧できること）
- 外部フォント・外部CSSライブラリの読み込みはしない（オフライン環境対応）
- JavaScriptフレームワーク（React/Vue等）は使わない（Vanilla JSのみ）
- LocalStorageへの個人情報保存は行わない

## 6. ワークフロー

### 6-1. 新規現場の追加
```
1. areas/_template.json をコピー → areas/area-XX.json にリネーム
2. JSON内の全フィールドを入力
3. 必要な画像を images/ に追加（命名規則: {エリアID}-{内容}.jpg）
4. admin.html でQRコードを確認・印刷
5. QRコードを現場に掲示
```

### 6-2. 既存コンテンツの更新
```
1. 該当の areas/area-XX.json を編集
2. updated フィールドを今日の日付に更新
3. updatedBy フィールドを更新者に変更
4. GitHub にプッシュ → GitHub Pages が自動デプロイ（約1分）
```

### 6-3. 画像の追加・差し替え
```
1. 画像を images/ フォルダに追加
2. ファイル名は {エリアID}-{内容の説明}.jpg 形式
3. JSONの image フィールドに相対パスを記載: "images/area-A1-shelf.jpg"
4. imageAlt フィールドに日本語の説明を記載
```

## 7. 実装制約・注意事項

### 絶対に行わないこと
- バックエンドサーバー・データベースの導入（無料枠超えのリスク）
- ユーザー認証機能の実装（閲覧はパブリック）
- 外部APIへの依存（サービス停止リスク）
- `node_modules` の git 追跡（`.gitignore` に必ず追加）

### パフォーマンス要件
- 初回ロード: 3G回線で3秒以内（LCP 3s以内）
- 画像サイズ: 1枚あたり最大500KB（JPEGで最適化）
- JSONファイル: 1現場あたり最大50KB

### ブラウザ対応
- iOS Safari 15以上
- Android Chrome 90以上
- （PCブラウザは考慮しなくてよい。スマホ専用）

## 9. ファイル生成優先順位

Claude Code がゼロから構築する場合、以下の順序で実装する：

1. **`areas/_template.json`** — データスキーマの雛形
2. **`areas/area-A1.json`** — サンプル現場データ（デモ用）
3. **`style.css`** — モバイルファーストのベーススタイル
4. **`app.js`** — JSON読み込み・DOM生成ロジック
5. **`index.html`** — メインビューア本体
6. **`admin.html`** — QRコード一覧・印刷ページ
7. **`qr-generator.html`** — QRコード生成（qrcode.jsライブラリ使用）
8. **`docs/update-guide.md`** — 非エンジニア向け更新手順書
9. **`docs/qr-print-guide.md`** — QRコード印刷・掲示手順書
10. **`README.md`** — セットアップ手順（エンジニア向け）

## 10. 品質チェックリスト

### コード生成後に必ず確認すること
- [ ] `index.html` がスマホ幅（375px）で正しく表示されるか
- [ ] `?area=` パラメータなしでアクセスした場合にエリア一覧が出るか

- [ ] JSONのパースエラー時にユーザーフレンドリーなメッセージが出るか
- [ ] `admin.html` で全現場のQRコードが一覧表示されるか
